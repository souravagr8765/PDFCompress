# How to Use the Logger in Other Scripts

To use the exact same logging setup in your other Python scripts (which also automatically integrates with your Loki dashboard monitoring via `compressor.log`), you just need to replicate the Python `logging` configuration.

Here are the step-by-step instructions.

## Step 1: Import the Required Modules
At the top of your new script, import the `logging` and `sys` modules.

```python
import logging
import sys
```

## Step 2: Add the Logging Configuration
Paste the following configuration block into your script before you start doing any work. This block tells the logger to output messages both to the console (`sys.stdout`) and to the same `compressor.log` file used by your main script.

```python
# Setup logging to match pdf_sync.py
LOG_FILE = "./compressor.log"

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s [%(levelname)s] %(message)s',
    handlers=[
        logging.FileHandler(LOG_FILE, encoding='utf-8'),
        logging.StreamHandler(sys.stdout)
    ]
)
logger = logging.getLogger(__name__)
```

> **Note**: If your new script might be executed from a different directory, you should provide the absolute path to `compressor.log` instead of `./compressor.log` to ensure all logs end up in the exact same file.

## Step 3: Use the Logger
Now you can use the `logger` object to record messages at various severity levels throughout your script.

```python
# Examples of logging
logger.info("This is a standard informational message.")
logger.warning("This is a warning message.")
logger.error("This is an error message.")
```

## Alternative: Create a Shared Logger Module
If you plan to create many scripts, it is best practice to put the configuration in a single file to avoid repeating code.

1. Create a new file called `shared_logger.py`:
```python
# shared_logger.py
import logging
import sys
import os

def get_shared_logger(name=__name__):
    # Use absolute path to ensure all scripts write to the exact same log file
    base_dir = os.path.dirname(os.path.abspath(__file__))
    log_file = os.path.join(base_dir, "compressor.log")
    
    logging.basicConfig(
        level=logging.INFO,
        format='%(asctime)s [%(levelname)s] %(message)s',
        handlers=[
            logging.FileHandler(log_file, encoding='utf-8'),
            logging.StreamHandler(sys.stdout)
        ]
    )
    return logging.getLogger(name)
```

2. Then, in any of your scripts, simply import it:
```python
from shared_logger import get_shared_logger

logger = get_shared_logger(__name__)

logger.info("Shared logger is working!")
```

By following these steps, any logs generated by your other scripts will append to `compressor.log` and automatically be correctly formatted.

## Step 4: Add the Loki Pusher Integration

The `loki_logger.py` script is designed to run in the background, watch the `compressor.log` file, and push new lines to your Loki dashboard. To make sure your script starts this pusher (just like `pdf_sync.py` does) and cleanly stops it, add the following code to your script:

1. Import the necessary modules at the top of your script if you haven't already:
```python
import os
import sys
import subprocess
import atexit
```

2. Add this setup block to your script, ideally near the beginning of your `main()` function or script execution. This will calculate the current log file size so the pusher only sends new logs, and then starts the `loki_logger.py` process in the background.

```python
# Global reference to the background loki logger process
_loki_process = None

def init_loki_logger(log_file):
    global _loki_process
    
    # Calculate where to start reading the log file (so we don't resend old logs)
    start_pos = 0
    if os.path.exists(log_file):
        start_pos = os.path.getsize(log_file)
        
    loki_url = os.environ.get("LOKI_URL")
    if loki_url:
        # Assumes loki_logger.py is in the same directory as this script
        base_dir = os.path.dirname(os.path.abspath(__file__))
        loki_script_path = os.path.join(base_dir, "loki_logger.py")
        
        if os.path.exists(loki_script_path):
            logger.info(f"Starting background Loki logger streaming to {loki_url}")
            try:
                _loki_process = subprocess.Popen(
                    [sys.executable, loki_script_path, log_file, str(start_pos)],
                    stdout=subprocess.DEVNULL,
                    stderr=subprocess.DEVNULL,
                    start_new_session=True # Runs independently
                )
            except Exception as e:
                logger.error(f"Failed to start Loki logger subprocess: {e}")
        else:
             logger.warning("loki_logger.py not found. Loki logging will not be available.")

# Call this to start the logger (pass the path to your log file)
init_loki_logger(LOG_FILE) # Assuming LOG_FILE = "./compressor.log" from Step 2
```

3. Add this cleanup function to make sure the Loki pusher is cleanly terminated when your script finishes. It signals the pusher to stop by writing a special message to the log.

```python
def cleanup():
    """Terminate background processes on exit."""
    global _loki_process
    if _loki_process is not None:
        logger.info("Terminating background Loki logger...")
        try:
             # Send termination signal via log file
             logger.info("LOKI_LOGGER_TERMINATE")
             _loki_process.wait(timeout=5)
        except subprocess.TimeoutExpired:
            _loki_process.kill()
        except Exception as e:
            logger.error(f"Error terminating Loki logger: {e}")
        logger.info("Loki logger terminated.")

# Register the cleanup function to run automatically when the script exits
atexit.register(cleanup)
```

With this complete setup, your new script will write logs to `compressor.log`, start the `loki_logger.py` process to push those logs to the cloud, and cleanly shut down the pusher when the script completes its work!
